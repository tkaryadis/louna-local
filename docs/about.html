Some context

Louna was developed λouna for my bachelor thesis at University of Crete.

I want to thank my supervisors Panagiotis Papadakos and Dimitris Plexousakis that allowed me to use Clojure, and were very nice to me.

Louna and other Projects

Since my aim was to develop something new, λouna was build from the ground up.
It is not based on any project and does not use code from anywhere else.
I wanted to make something my own and then see alternatives and better ways.

I always liked prolog,λouna's syntax is heavily influenced by SPARQL and the inspiration comes from  cascalog.
But it doesn't use code from any SPARQL processor or from the cascalog processor. 

The template for the web-site is a bootstrap template.

External libraries
λouna has 2 code dependencies except clojure, 

An analyzer for Clojure code [org.clojure/tools.analyzer.jvm "0.7.2"].
In the future it might be removed though, since currently its only use is to tell you that x is a local name
in queries of the type (defn f [x] (q (:person.name ?name x)) 
(found on  stackoverflow).

An rdf converter RDF2RDF to convert .rdf and .ttl files to .nt files that louna can read.

Test examples
Most of the provided examples are from the book Learning SPARQL, 2nd Edition by Bob DuCharme.
The book gave alot of examples in SPARQL and helped alot with testing. Currently λouna can do almost everything
up to chapter 5 (until page 185); the rest has to do with SPARQL update mostly which is currently not supported.
λouna can do things that SPARQL can't do because λouna is programmable.
For example queries with arguments coming from a function, or construct binds to relations, or data in memory etc.)

I used also example queries from a W3C web page.

I wrote also my examples to support the Documentation with simple examples.

In test folder in louna source code,there are many queries.

The best way to test louna is with tables,let louna read a table(see about tables in Input Output),save it to a DB 
(c {:c-out ["mydb"]}
   (?id ?name)
   (?id ?lastname)
   ....
   (q (:table (:person ?id ?name ?lastname)))

You can save to mydb multiple tables and then do queries to them.
